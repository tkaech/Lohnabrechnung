Windows-Teststart Payroll.Desktop
================================

Ziel
----
Diese Version ist fuer einen ersten Windows-Testlauf gedacht.
Sie verwendet SQLite und legt die Datenbank beim ersten Start automatisch an.


Standardverhalten
-----------------
- Die Anwendung liest standardmaessig `appsettings.json`.
- Der dort konfigurierte Datenbankpfad ist:
  `{LocalAppData}/PayrollApp/payroll.windows-test.db`
- Unter Windows wird `{LocalAppData}` typischerweise auf
  `C:\Users\<Benutzer>\AppData\Local`
  aufgeloest.
- Fuer diesen Windows-Testlauf werden beim ersten Start Testdaten angelegt,
  wenn die Datenbank noch leer ist.

Beispiel:
`C:\Users\<Benutzer>\AppData\Local\PayrollApp\payroll.windows-test.db`


Erster Start
------------
1. `Payroll.Desktop.exe` starten
2. Falls die Datenbank noch nicht existiert:
   - wird die SQLite-Datei angelegt
   - werden EF-Migrationen ausgefuehrt
   - werden Konfigurationsdaten initialisiert
   - werden Test-Mitarbeitende fuer den Windows-Test angelegt
3. Wenn die Datenbank bereits existiert, bleibt sie unveraendert erhalten


Weitere Starts
--------------
- Die bestehende Datenbank wird weiterverwendet
- Bestehende Daten werden nicht geloescht
- Die Anwendung fuehrt nur noch erforderliche Migrationen aus


DB-Pfad ueberschreiben
----------------------
Es gibt drei Moeglichkeiten:

1. Startparameter
`Payroll.Desktop.exe --db-path="C:\Pfad\zur\payroll.windows-test.db"`

2. Environment-Variable
`PAYROLLAPP_DATABASE_PATH`

Optional fuer Testdaten:
`PAYROLLAPP_SEED_TESTDATA=true`

3. Appsettings
- `appsettings.json`
- `appsettings.Development.json`


Environment setzen
------------------
Optional kann das Environment gesetzt werden:

`Payroll.Desktop.exe --environment=Production`

Moegliche Werte:
- `Development`
- `Production`
- `Test`


Beispiele
---------
Standardstart:
`Payroll.Desktop.exe`

Start mit explizitem Testpfad:
`Payroll.Desktop.exe --db-path="C:\Users\<Benutzer>\AppData\Local\PayrollApp\payroll.windows-test.db"`

Start mit anderem Environment:
`Payroll.Desktop.exe --environment=Production`


Wichtige Hinweise
-----------------
- `appsettings.json` liegt im gleichen Ordner wie die Anwendung
- Der Ordner fuer die Datenbank wird bei Bedarf automatisch angelegt
- Die Anwendung verwendet SQLite
- Fuer diesen Testlauf gibt es noch keinen Installer


Fehlerbehebung beim ersten Start
--------------------------------
Wenn der Start fehlschlaegt oder eine unvollstaendige Test-Datenbank vorliegt:

1. Anwendung schliessen
2. Datei loeschen:
   `C:\Users\<Benutzer>\AppData\Local\PayrollApp\payroll.windows-test.db`
3. Anwendung erneut starten

Zusatz:
- Beim erneuten Start werden Datenbank und Testdaten frisch aufgebaut
- Falls weiterhin ein Fehler auftritt, Logdatei pruefen:
  `C:\Users\<Benutzer>\AppData\Local\PayrollApp\startup-error.log`
