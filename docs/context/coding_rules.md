# Coding Rules

## Grundprinzipien

- DRY (keine doppelte Logik)
- Single Source of Truth
- Wiederverwendung vor Neuerstellung

## Architektur

- keine Businesslogik in UI
- keine DB-Zugriffe in UI
- klare Verantwortlichkeiten

## Daten

- Geld: decimal
- keine hardcodierten Werte
- Regeln versionierbar

## Qualität

- Tests für kritische Logik
- klare Namen
- kleine Klassen
